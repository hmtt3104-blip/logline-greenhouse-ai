# Contributing

## Purpose

This repository is part of Logline, a personal engineering laboratory and public experiment journal.

Contributions should improve clarity, reproducibility, safety, or the quality of the engineering record.

## Useful contributions

- documentation fixes;
- setup notes;
- safety notes;
- experiment records;
- sanitized examples;
- tests or small focused code fixes;
- reproducibility improvements.

## Rules

- Do not include secrets.
- Do not include production configs.
- Do not include private IPs or live deployment details.
- Do not include personal images or EXIF metadata.
- Do not add APKs, build outputs, or binary artifacts.
- Keep changes focused and documented.

## Experiment records

Experiment records should use the Logline structure:

- Problem
- Hypothesis
- Experiment
- Environment
- Hardware
- Software
- Data
- Results
- Failures
- Lessons
- Next Question
- Status
- Trust level
- Links

## Pull request checklist

- [ ] The change is scoped.
- [ ] README or docs are updated if behavior changed.
- [ ] No secrets or production details are included.
- [ ] Public-readiness risks are marked if found.
- [ ] Experiment records are added or linked when relevant.
